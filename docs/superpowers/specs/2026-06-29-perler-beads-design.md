# 拼豆图纸生成器 — 设计文档

**日期**: 2026-06-29
**状态**: 设计中

---

## 1. 概述

网页工具，用户上传图片 → 自动转换成拼豆图纸。图纸上每个格子用品牌编号标注豆子颜色，附带豆子用量清单，支持导出和打印。

**MVP 阶段**：前端 Canvas API 处理图片 + 后端 FastAPI 服务基础架构（预留 AI 接口）。


数据来源：`beadcolors.eremes.xyz`（Beadifier 项目维护），内嵌为 JSON。

## 6. 颜色匹配算法

```
MVP 阶段: CIEDE2000 感知距离（纯 JS 实现，无后端依赖）

对于网格中每个像素 (r, g, b, a):
  // 1. 空白检查
  if a < 128:                         → 空白
  if 用户指定跳过色且 ΔE ≤ 3:         → 空白

  // 2. 智能减色（MVP 用 K-Means，Phase 2 用 AI）
  if 保真度滑块 < 100%:
    将像素颜色聚类到 N 种代表色（N 由滑块决定）
    保护：主体色、边缘色、大面积色不丢失

  // 3. CIEDE2000 匹配
  bestColor = null, bestDist = Infinity
  for palette 中每种颜色:
    dist = CIEDE2000(r, g, b, pr, pg, pb)  // 感知色差
    if dist < bestDist:
      bestDist = dist, bestColor = color
  记录 bestColor 到网格
```

- Web Worker 中执行，主线程不阻塞
- 相同 RGB 值缓存匹配结果（Map 查找）
- CIEDE2000 比 RGB 欧几里得距离更接近人眼感受

## 8. 图纸格式

- **网格**：彩色背景 + 品牌编号文字（浅底深字/深底浅字）
- **底板线**：每 29 格一条粗线，标记物理底板边界
- **空白格**：留空，仅显示浅灰边框
- **豆子清单**：表格含编号、颜色名、色样圆点、数量；点击某行触发单色高亮
- **单色高亮**：图纸上仅该颜色格子变亮，其余半透明灰
- **打印分页**：超出 A4 的图纸自动拆分多页，每页带对齐标记

## 12. 边界与限制

- 最大网格 500×500（Web Worker 保证不卡 UI）
- sRGB 色彩空间
- MVP 不实现抖动算法
- 移动端响应式：三栏 → 上下堆叠
- AI 功能需后端代理调用外部 API，MVP 阶段接口预留但不实现

## 13. 性能策略

| 策略 | 说明 | 效果 |
|------|------|------|
| Web Worker | 所有颜色计算放在 Worker 线程 | 主线程零阻塞 |
| 渐进渲染 | 先出 1/4 分辨率预览 (<30ms)，后台替换高清 | 用户感知「秒出」 |
| 颜色缓存 | Map<RGB, BeadColor>，相同 RGB 只算一次 CIEDE2000 | 命中率 60-90% |
| 滑块防抖 | 保真度滑块 150ms debounce，拖动时不计算 | 省 95% 无效计算 |
| 请求去重 | 快速切换品牌/参数时，取消前面未完成的计算 | 不浪费算力 |
| 品牌推荐懒加载 | 先用当前品牌出结果，另外两个品牌后台算 | 推荐面板打开即就绪 |

## 14. 品牌推荐

上传图片后，自动对三个品牌色表各跑一次 CIEDE2000 匹配，比较平均 ΔE，推荐误差最小的品牌。用户可选择接受推荐或手动切换。

## 15. 颜色精准度保障（四层）

1. **CIEDE2000 感知色差** — Lab 色彩空间 + 亮度/色度/色相修正，数值=感受
2. **智能减色** — 加权 K-Means：主体×3、边缘×2 保护，面积<1% 才允许合并；噪点清洗 + 近距合并（ΔE<3 且占比<5%）防冗余
3. **用户安全网** — 滑块下限 3 色，ΔE>5 黄警 ΔE>10 红警，随时可撤回
4. **AI 预处理**（Phase 2） — 只做去背景/锐化等确定性操作，不动主体像素
